# Contributing to Anidaku

Thank you for considering contributing to Anidaku! This document provides guidelines and instructions for contributing to the project.

## Code of Conduct

- Be respectful and inclusive
- Accept criticism gracefully
- Focus on the code, not the person
- Help others learn and grow

## How to Contribute

### Reporting Bugs

1. Check existing issues to avoid duplicates
2. Provide clear title and description
3. Include:
   - Device model and Android version
   - Steps to reproduce
   - Expected vs actual behavior
   - App version
   - Screenshots/logs if applicable

### Requesting Features

1. Search existing feature requests
2. Provide clear use case and benefits
3. Include mockups or examples if helpful
4. Discuss alternatives if relevant

### Submitting Code

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/amazing-feature`
3. Commit changes: `git commit -m 'Add amazing feature'`
4. Push to branch: `git push origin feature/amazing-feature`
5. Open a Pull Request

## Pull Request Guidelines

### Before Submitting

- [ ] Code follows project style guide
- [ ] Tests added/updated
- [ ] No console errors or warnings
- [ ] Commits are clear and descriptive
- [ ] Branch is up-to-date with main

### PR Description

Include:
- What problem does it solve?
- How does it work?
- Any breaking changes?
- Screenshots/demo links
- Testing instructions

## Development Workflow

### Branch Naming

- `feature/description` - New features
- `fix/description` - Bug fixes
- `docs/description` - Documentation
- `refactor/description` - Code refactoring
- `chore/description` - Build, deps, etc.

### Commit Messages

```
type: short description (50 chars)

Optional longer explanation (72 chars per line)

Breakin Changes:
- List breaking changes

Fixes: #123
```

### Code Review

- Constructive feedback only
- Request changes politely
- Approve when satisfied
- Merge after approval

## Getting Help

- Check existing issues and discussions
- Review documentation
- Ask in pull request comments
- Create discussion for architecture questions
